# RFC

## Контекст

- Необходимо разработать схему кодирования для алфавита, состоящего из символов ALPHABET
- Главное требование: схема должна быть максимально экономичной по памяти
- Подсказка: необязательно кодировать все символы "как есть", как и необязательно делать фиксированный размер на символ

## Вариант 1. Фиксированная длина кодов, 8 бит на символ

Плюсы:

- код самый простой
- у каждого символа всегда одинаковая цена
  - так как известны границы символов, то легко искать подстроку в закодированном тексте
- декодирование тоже простое

Минусы:

- частые буквы стоят столько же, сколько редкие символы
- это хуже по размеру, чем переменная длина кодов без выравнивания, потому что по сути остается много пустых битов в каждом байте

## Вариант 2. Переменная длина кодов, но частоты захардкожены

Плюсы:

- компактнее по памяти, потому что поток битов пишется плотно без выравнивания по байтам
- частые символы дешевле редких

Минусы:

- код сложнее, потому что нужно писать поток битов, а не байтов
- схема заточена под среднестатистический текст на русском языке, а не конкретный
  - на нетипичных текстах с большим количеством редких символов (код, формулы), может работать хуже, чем вариант 1
- не известны границы символов, нужно сначала декодировать поток, чтобы найти подстроку

## Вариант 3. Переменная длина кодов и динамические веса, Huffman

Плюсы:

- считает частоты символов под конкретный текст
- самый компактный результат по размеру результата из трех

Минусы:

- я не знаю алгоритм Хаффмана
- самый сложный вариант
- нужно сначала пройти по тексту, чтобы посчитать частоты, потом построить дерево, потом уже кодировать
- оверхед на хранение дерева в закодированном тексте, чтобы потом декодировать
  - съест выигрыш от оптимизации на коротких текстах
- не известны границы символов, нужно сначала декодировать поток, чтобы найти подстроку

## Решение

В итоге выбрал вариант 2, как компромисс между сложностью и эффективностью по памяти. По сути это похоже на сжатие текста. Чтобы сделать этот вариант более похожим на кодировку, можно было бы добавить индекс смещений с каким-то сэмплированием, типо на каждый 100-й символ. Для учебной задачи думаю более, чем достаточно

За основу взял [частотность букв русского языка с Википедии](https://ru.wikipedia.org/wiki/Частотность)
